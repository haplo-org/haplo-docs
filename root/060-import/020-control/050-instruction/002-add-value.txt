title: add-value
link_as: keyword
--

The @add-value@ instruction adds a constant value to a *Name* in a *Destination*. It is called _add_ value because if the *Name* is a multi-value, then it will add the value rather than replacing it.

|@action@|@"add-value"@|
|@destination@|The destination to store the value in.|
|@name@|The Name within the destination to store the value in.|
|@value@|The constant value.|

The value specified in the control file uses the normal value conversions, so all the [node:import/control/value-conversion:value conversion properties] can be used.

*WARNING*: To replace the value in a multi-value Name, you need to precede this instruction with a [node:import/control/instruction/remove-values] instruction. (Except if you're writing a control file for a user sync.)


h2. Example

<pre>language=json
{
    "action": "add-value",
    "destination": "profile",
    "name": "dc:attribute:type",
    "value": "std:type:person:staff"
}
</pre>


h2. Example: replacing multi-value

To set the value of a multi-value Name, replacing any values that are already set, use a pair of @remove-values@ and @add-value@ instructions.

<pre>language=json
{
    "action": "remove-values",
    "destination": "profile",
    "name": "std:attribute:job-title"
},
{
    "action": "add-value",
    "destination": "profile",
    "name": "std:attribute:job-title",
    "value": "Director of Research"
}
</pre>
