title: field
link_as: keyword
--

The @field@ instruction copies a value from the input *Record* to a *Name* in a *Destination*.

As it is the most common Instruction, the @action@ property may be omitted.

|@source@|The property in the input Record.|
|@destination@|The destination to store the value in.|
|@name@|The Name within the destination to store the value in.|
|@multivalue@|(Optional) If @true@, then the value is added to a multi-value name instead of replacing any existing value.|

In addition, [node:import/control/value-conversion:value conversion properties] can be added, especially [node:import/control/value-conversion/filters:value filters], [node:import/control/value-conversion/data-type/date:date formats], [node:import/control/value-conversion/generic-value-mapping:value mapping], and [node:import/control/value-conversion/data-type/ref:ref mapping].


h2. Example: single value

To set @username@ within the @user@ destination using the value of the @ID@ property in the input Record:

<pre>language=json
{
    "source": "ID",
    "destination": "user",
    "name": "username"
}
</pre>


h2. Example: mapping to ref

To set an organisational unit in the user's @profile@, [node:import/control/mapping:mapping a code] from the input:

<pre>language=json
{
    "source": "orgUnit",
    "destination": "profile",
    "name": "example:attribute:organisational-unit",
    "refMapping": "organisational-unit"
}
</pre>


h2. Example: working with multi-values

To set the value of a multi-value Name, replacing any values that are already set, use a @remove-values@ instruction. This example sets the job title to "Researcher" if depending on a value in the input file, then additionally adds any job title from the input file.

<pre>language=json
{
    "action": "remove-values",
    "destination": "profile",
    "name": "std:attribute:job-title"
},
{
    "source": "ROLE",
    "action": "if-value-one-of",
    "values": ["RESEARCHER"],
    "then": [
        {
            "action": "set-value",
            "destination": "profile",
            "name": "std:attribute:job-title",
            "value": "Additional job title",
            "multivalue": true
        }
    ]
},
{
    "source": "JOB_TITLE",
    "destination": "profile",
    "name": "std:attribute:job-title",
    "multivalue": true
}
</pre>
